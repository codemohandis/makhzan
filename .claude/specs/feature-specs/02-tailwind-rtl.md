# Spec: Tailwind RTL Configuration

## 1. Context & User Story (Human Layer)

- **SRS Reference:** NFR-01 (RTL/Accessibility), NFR-02 (Performance), FR-05 (Language Toggle — font infrastructure), SRS §2.2 Tech Stack, §8 Phase 1 items 2 & 4
- **Overview:** Install and configure Tailwind CSS with the `tailwindcss-rtl` plugin, initialize Shadcn/UI with a custom "Digital Manuscript" theme, load both Arabic-script fonts via `next/font/google`, and create a `GlobalContainer` layout primitive — establishing the complete styling foundation that every subsequent UI component will inherit.
- **User Story:** "As a developer, I want a fully configured RTL-first styling system so that every component I build automatically gets logical CSS properties, the correct Urdu/Farsi font, and the 'Soft Paper' manuscript aesthetic without any per-component setup."
- **Depends on:** Step 01 — Next.js App Router Scaffold

## 2. Technical Blueprint (Agent Layer)

### Interface & Contract

| Interface | Method | Path / Action | Access | Behaviour |
| :--- | :--- | :--- | :--- | :--- |
| Page | GET | `/` | Public | Renders with soft paper background (`#F9F7F2`), Noto Nastaliq Urdu font, `line-height: 2.0` |

### Shadcn CSS Variable Override (`:root` in globals.css)

```css
:root {
  --background: 40 33% 97%;         /* #F9F7F2 soft paper */
  --foreground: 20 14% 10%;         /* #1C1917 dark ink */
  --primary: 215 100% 19%;          /* #002147 Oxford Blue */
  --primary-foreground: 40 33% 97%;
  --accent: 345 100% 25%;           /* #800020 Burgundy */
  --accent-foreground: 40 33% 97%;
  --border: 30 10% 82%;
  --ring: 215 100% 19%;
  --radius: 0.375rem;
  --font-body: var(--font-nastaliq);
}

html[lang="fa"] {
  --font-body: var(--font-vazirmatn);
}
```

### Typography Defaults

```css
html[lang="ur"] body { line-height: 2.0; letter-spacing: 0; }
html[lang="fa"] body { line-height: 1.8; letter-spacing: 0; }
```

> **Why `letter-spacing: 0`:** Nastaliq is a connected script — any positive tracking visually breaks glyph ligatures.

### RTL Checklist

- [ ] All layout classes use logical properties (`ps-*`, `pe-*`, `ms-*`, `me-*`, `text-start`, `text-end`)
- [ ] No `left-*`, `right-*`, `ml-*`, `mr-*`, `pl-*`, `pr-*`, `text-left`, `text-right` in new or modified files
- [ ] `GlobalContainer` tested with `lang="ur"` and `lang="fa"` on `<html>`
- [ ] `/fix-rtl` run on `GlobalContainer.tsx` — 0 replacements

### Logic Scenarios (Gherkin-style)

- **Scenario:** Urdu font loads correctly
  - **Given:** `<html lang="ur">` is set in layout
  - **When:** Browser renders `localhost:3000`
  - **Then:** DevTools Computed panel shows `font-family` resolving to Noto Nastaliq Urdu; `line-height` = 2

- **Scenario:** Farsi font switches via `lang` attribute
  - **Given:** Developer sets `<html lang="fa">` (simulating language toggle)
  - **When:** Page renders
  - **Then:** DevTools shows Vazirmatn font and `line-height` = 1.8; layout remains RTL

- **Scenario:** No LTR classes in source
  - **Given:** Feature is fully implemented
  - **When:** `grep -r "text-left\|ml-\|mr-\|pl-\|pr-" src/` is run
  - **Then:** Zero matches

- **Scenario:** Tailwind utilities available
  - **Given:** `tailwind.config.ts` is configured with RTL plugin
  - **When:** A component uses `ps-4` (padding-inline-start)
  - **Then:** Correct padding is applied on the start side in both Urdu and Farsi layouts

### Hard Constraints (The Guardrails)

- **Never** use `left-*`, `right-*`, `ml-*`, `mr-*`, `pl-*`, `pr-*`, `text-left`, `text-right` — use logical equivalents only
- `tailwind.config.ts` must include `require("tailwindcss-rtl")` in plugins array
- `next/font/google` is mandatory for both fonts — no CDN `<link>` tags (causes layout shift)
- `--font-body` CSS variable must be the single source of truth for font switching — no inline `font-family` styles
- Shadcn components path: `src/components/ui/` — never outside `src/`
- `src/lib/utils.ts` is auto-generated by Shadcn CLI — do not hand-write it
- `SUPABASE_SERVICE_ROLE_KEY` not needed in this step — do not reference it
- TypeScript strict mode — all new files must pass `npx tsc --noEmit` with 0 errors
